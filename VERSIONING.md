# Documentation Versioning

This repo uses a docs-content versioning model:

- `apps/docs/content/docs/` — latest documentation
- `apps/docs/content/docs/v1/` — versioned snapshots for first public release
- future snapshots can be created in `v2`, `v3`, and so on

Suggested release tags:

- `docs-v1.0.0-beta`
- `docs-v1.0.0`
- `protocol-v1.0.0-alpha`
